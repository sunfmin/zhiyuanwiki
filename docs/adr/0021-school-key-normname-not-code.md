# 院校主键改用归一化校名，废弃「稳定校代码」——院校代号非跨年跨源稳定键（订正 ADR-0003）

ADR-0003 立院校为主干稳定实体，称其「由**稳定的校代码**标识」。这个前提经黑龙江数据实测**不成立**：
「广州大学在黑龙江消失、且其 2026 招生被错挂到深圳大学页面」的根因，正是把院校代号当成了跨年、跨源的稳定键。
本 ADR 订正主键，并把全站早已事实使用的锚点（`core.NormName` 归一化校名）扶正。

## 订正（实测推翻「稳定校代码」）

院校代号是**逐年重排的位次码**，三重不稳：

1. **老/新高考换本**。黑龙江 2024 起新高考，2023 是老高考、另一套代号本。plan(2026) 对 score 各年
   「同代号异校名」冲突：**2023=251 / 2024=29 / 2025=6**——所谓「261 处全面错位」几乎全由 2023 混入造成，
   且是整齐**位移**（plan `1004`=人大、2023 `1004`=北航，全表偏移），非随机错乱。
2. **相邻年份也会因插入学校而位移**，且**同校多代号**：一所校在同一份计划里按普通/中外合作/专项分持多码
   （黑龙江大学 `1408,1413,6001`；哈尔滨理工大学 `1409,1410,6002`，`60xx` 为中外/专项块）。按代号建页会把
   一所学校**碎成多页**。
3. **万师兄的「院校代号」并非考试院官方代号**，是其自编内部编号且跨年不自洽：官方深圳大学黑龙江 2023=1222、
   2024=2044，我们数据里 2023=2046、2024/25=2044（2024 起才与官方对上）。**代号不可作跨源键。**

反证：把校名过 `NormName` 后，跨源（plan2026↔score2025）**零**纯写法误配（35 处未匹配 = 6 改名 + 29 真·新招生校），
源内跨年**零**写法漂移。**校名稳、代号不稳。**

## 决定

1. **主键 = 归一化校名 `core.NormName`，粒度=报考实体（校区级）**。`NormName` 保留「(苏州校区)」等后缀，
   天然把 `中国人民大学(苏州校区)` 与 `中国人民大学` 分成两个独立院校（投档线、城市各异，考生分开填）。
   `core.BaseName`（去括号后缀）得法人母体，**仅**用于把 985/211 等母体标签继承给校区，不作主键。
   普通/中外/专项的多代号在同一校名页下呈现为多个**院校专业组**（承 ADR-0003：组是单年视图，不是实体）。

2. **院校全集 = `NormName(plan 全年) ∪ NormName(score 全年)`**，两类空态页都建：
   plan-only（广州大学等 29 所，2026 首招无历史）标「暂无历年录取数据」；score-only（今年未招）标
   「2026 未在黑龙江招生」。修复了「学校一览只从 score 建、plan-only 校消失」的旧 bug（`yuanxiao_db.go:30`）。

3. **历史叶子只按 `NormName` 挂到实体，代号一律不参与 identity**；尤其 2023 老高考代号本只按名挂接，
   从根上斩断 251 处误连。2026 组接历年专业分数线仍按「院校+专业名」等效挂接（承 ADR-0003），不按码套。

4. **改名靠推导、不手维护别名表**：相邻新高考年份「代号相同、名字变了」即改名候选，按名干相似度分三档——
   高度重合（湖州师范学院→大学）自动接；零重合（老高考位移误连）自动拒；弱相似（转设，大连理工大学城市学院→
   大连工程学院）**打日志交人工**。转设默认接历史并在页面标注「原名 XX（YYYY 转设）」，语义变更透明化。

## 后果

- `school_attr`（按 `(prov, school_code)`）与任何按代号的属性挂接需改为按 `NormName`，与全国 `school`/
  `major_catalog` 表（本就按 `name_norm`=`NormName`）归一到同一把键。
- ADR-0003 的「稳定校代码」表述作废，其余（院校/专业为主干、组为单年视图、按名挂接）继续有效。
- 校名当键的两个失效模式（改名、写法差异）分别由「推导式改名识别」与「`NormName` 归一」兜住，均无需手同步。
