# 网站形态：静态数据百科 + 方法论，客户端做位次定位

本站定位为"数据驱动的院校/专业百科 + 方法论指南"的混合 wiki：每所院校、每个院校×专业有数据驱动的常驻页面（历年投档线/位次/招生计划走势），外加一批方法论文章（平行志愿、等效位次法、冲稳保、防调剂）。

决定**以 Astro 静态站为主、无服务端**：所有页面在 build 时由预生成数据渲染成静态 HTML；唯一的动态能力——"输入位次→看冲稳保"——用一个**客户端 island** 在浏览器本地对预生成 JSON 做筛选与分档完成。

放弃了"交互式定位工具/web app"（需要服务端或更重前端）与"纯方法论 wiki"（浪费了 2020–2026 的结构化数据金矿）两个方向。代价：个性化能力受限于客户端能加载的数据量；好处：零后端运维、零隐私/备案负担、托管极简。
