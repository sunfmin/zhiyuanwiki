# 冲稳保按把握比值分档，不按位次邻域凑数

定位结果的冲/稳/保**按访客位次 V 与专业等效位次 R 的比值 `V/R` 分档**，每档只显示真正落入区间的条目，**不再为了凑满数量而把远档拉进来**。

**比值阶梯（一处定义，两处消费）**：`够不着 >1.15`、`冲 1.02–1.15`、`稳 0.90–1.02`、`保 0.75–0.90`、`过保 ≤0.75`。落在 `dingwei.ts` 的单个 `classify(V,R): Bucket|null`（`null`＝无效输入，不是某一档）。定位页 `Locator.tsx` 与院校页角标 `SchoolBadges.tsx` **都消费这一个函数**，不再各算各的。

**这是对上一版「按位次邻域取档」的回退**（commit da3e708 那条「冲稳保改按位次邻域取档」）。邻域法以你的位次为锚，固定取上/中/下各约 100 条充当冲/稳/保——列表永远丰满，但会把「高你 13000 位（30 万考生里差 >13%）」这种**根本搏不到**的专业标成「冲」。把握标签因此失真：定位页说「冲」，院校页角标（一直用比值的 `classify`）却说「够不着」，两个界面自相矛盾，也违背术语表「冲＝录取位次略好于本人」的定义。

**主档为什么放弃「保证 100」**：邻域法把 100 凑进了**冲稳保本身**——这是病根。同一个窗口同时干了两件相反的事：**给稀疏档凑数** 又 **给密集档截断**。拆开后主档只装真档：稀疏（如筛了计算机+985）就有几条显几条；密集才软上限 + 展开更多，纯防刷屏、绝不向上填满。短而诚实的冲，胜过长而灌水的冲。

**但「保证 100」作为浏览量诉求本身不错，故把它移到远档**：真实冲/保不足时，用够不着/过保把**冲列、保列各补齐到约 100 条**。凑数因此只发生在**被降级、被标记的"仅供参考"区**，主档的把握标签永远不被污染。这是「保证 100」与「不误标」的诚实和解。

**为什么用比值，不用人数占比或绝对位次差**：比值是尺度无关的，天然吃进了「你所在位置的分数密度」——同样的绝对位次差，在低位次区（分数密集）够得着，在高位次区（分数稀疏）就是幻想。「30 万考生差 13000」这种按人数占比的直觉，比值已经覆盖（且不必引入本站数据里根本没有的全省考生总数）。绝对位次差则跨科类/跨省/跨位置都不成立。

**两端做「远档」，不误标**：超出冲的叫**够不着**，低于保的叫**过保**（白白浪费位次，新加的 `0.75` 下限就是为它划界）。两者在对应列末尾折叠成「仅供参考」展开块——把本列补齐到约 100 条（真实档已 ≥100 则不补），**视觉上与真实候选明显区分**（虚线粗分隔 + 置灰 + 灰度），绝不与真实候选争位。

**取舍与边界**：
- 阈值 `1.02 / 1.15 / 0.90` 沿用院校页角标已上线的值——改它们会连带重标所有历史角标。`1.15` 作「搏一把」上限偏宽，留待**回退修复上线后**若发现冲档顶部仍有杂质再收紧（届时有真实列表可判断，胜过现在拍脑袋）。
- **等效分**随每条以「约/≈」展示，仅作辅助感知，位次仍是排序与分档的唯一基准（见 CONTEXT.md「等效位次」「冲/稳/保」）。
- **不显示录取概率百分比**：单年位次无法诚实推出「73%」之类精度，假精度会反噬「位次为准」的根基；档位 + 位次差 + 等效分差已是诚实的上限。
- 代价：筛窄时**主档**会比以前显示得少（少而准，有意为之）；但远档把整列补齐到约 100，想多看仍有"仅供参考"可展开。
