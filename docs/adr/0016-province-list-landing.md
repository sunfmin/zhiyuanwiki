# 省份列表落地页：全 31 省一张表，省情全部出自构建管线（不开手工事实源）

根路径 `/` 此前只做 302 跳转到默认省份（黑龙江，ADR-0009），没有"先看全国、再选省"的入口。
随着接入到 6 省（黑龙江/浙江/江苏/湖南/四川/安徽），需要一个**省份列表落地页**：一行一省、可搜可排，
点进去到 `/[prov]/`（位次定位）。

设计时定了几个会咬人的取景判断，逐条记下原因。**核心约束：省情数据也只走构建管线（ADR-0004/0014），
不另开手工维护的第二事实源。**

## 决定

1. **列全 31 省，不只列已上线的省。** 已上线省进完整对比表（行行可点）；未上线省**置灰不可点、状态标
   「敬请期待」**。理由：河南/山东/广东这类大省尚未上线，整省消失会让该省考生以为本站永不支持。
   **固定顺序**（构建期定死，无搜索）：**已上线 6 省置顶**（按高考人数降序）→ 敬请期待 25 省**按拼音
   首字母 A→Z**。无搜索时靠拼音序线性扫到自己的省。"已上线"以**该省有投影数据**判定，不硬编码
   （管线就绪省 > 已配置省——DB 里已有重庆等的一分一段）。

2. **两轴分区：省情 vs 本站收录。**
   - **省情**（真实世界事实）：高考人数、物理特控线、新高考模式、**本省院校数**。
   - **本站收录**（覆盖元数据）：**招生院校（可报考）数**、数据条数、覆盖年份、名校数。

3. **"院校数"拆成两个一等口径**（见 CONTEXT.md）：
   - **招生院校** = 本省有录取记录、可报考的全国高校（`schools.json` 行数，1000+）。**受收录年份污染**
     ——四川/安徽仅 2025，被压到 463/472，不是真相。归"本站收录"轴，旁边必须配"覆盖年份"作口径。
   - **本省院校** = 校址在本省的高校，来自全国 `school.province`（江苏 175、河南 185）。**不受收录影响**，
     是省情事实。归"省情"轴。两数并列易混，靠分区 + 标签区隔。

4. **省情全部 pipeline-native，不开手工源**（这是对本 ADR 初稿的纠正——初稿曾打算建手工省情表）：
   - **高考人数 = 一分一段最大累计**（`MAX(yifenyiduan.cum)` per 省×年×科类，本科批最新年；3+1+2 省可合计
     物理+历史）。DB 现成。**口径必须诚实标注：是"参加统一高考排名的人数"，不含高职单招/中职对口，
     ≈ 但不等于"报名人数"。**
   - **物理本科线**（实测修正：源给的是本科线，非特控线）：源一分一段表本就带「批次 / 控制线(分)」列
     （`hn/parse.go` 等读了「批次」做本科过滤，但**丢了控制线**）。**新增解析+入库+投影**即得。实测各省
     本科批控制线值（江苏 2025 物理 463、历史 482；湖南物理 405…）落在**本科线**区间，**不是**更高的
     特殊类型招生控制线（特控线）——源一分一段只有一列「控制线」，对本科批行即本科线。故列名定为「本科线」，
     标注物理类本科批最低控制线。浙江综合科、黑龙江（走 core 解析路径未采）显「—」。
   - **本省院校数**：`school.province` 全国表，**全 31 省可填**（含未上线省）。
   - **新高考模式**：从 `provinces.ts` 取。
   - **录取率不做**：各省份源数据里没有、推不出，做它就得开第二源，与单一源原则冲突。

5. **未上线省行只有"本省院校数"有值。** 高考人数/特控线都依赖该省一分一段，未导入源数据的省（河南/山东）
   推不出。故未上线行 = 省名 + 本省院校数（全国表）+「敬请期待」，其余列空、不可点。

6. **纯静态页，无客户端 JS。** 做成普通 Astro 页（`getStaticPaths`，与其余页同构），**不引 Preact island**
   ——无搜索框、无交互列排序，顺序构建期定死（见决定 1）。响应式双形态靠纯 CSS：桌面完整分组宽表，
   手机每省一张卡片。"对比"是固定序对照，不是可重排表——这是有意的简化（落地页只为选省，不是数据分析台）。

7. **修 `is985`/`is211` 过标 bug。** 全国 985 仅 39 所，但 `school` 表按校名挂接后多省 `is985` 计到 50
   （疑似分校区/独立学院误标）。名校列要显真值，须先在管线侧收口标签逻辑，否则"50 > 39"穿帮、且与
   "四川 12"并列会被误读成"四川名校少"（实为覆盖年份少）。

## 影响

- 根路径从"纯跳转"升级为有内容的落地页；`/[prov]/` 仍是各省主页。
- **不新增任何手工事实源**——省情全部从既有 SQLite staging（ADR-0014）派生，守住 ADR-0004 单一源。
- 触发的管线改动：
  - `yifenyiduan` 解析**新增「控制线」**：动 `internal/{hlj,zj,js,hn,sc,ah}/parse.go` 读控制线 + `store`
    schema 加列（或建 `batch_line` 小表）+ 落地页投影。回归护栏护住已提交数据。
  - 高考人数投影：`MAX(cum)` per 省×年×科类（LoadTotals 已有同款逻辑可复用）。
  - 本省院校投影：按 `school.province` 分省计数，覆盖全 31 省。
  - **`is985`/`is211` 过标修复**（影响所有省的名校计数，改已提交数据，需回归护栏）。
- 落地页"已上线/敬请期待"由数据存在性派生，新省自动转态。

## 给未来的人

- 接新省 → 落地页该省**自动从敬请期待转已上线**，省情列（高考人数/特控线）随其一分一段入库**自动有值，
  无需补任何手工数据**。这正是"走管线"的回报。
- 永远别拿"招生院校数"跨省比规模：它是覆盖度指标，不是省情。比"这省多大"用本省院校数或高考人数。
- 高考人数是**统考排名人数**，不是报名人数，标注别糊弄。
- 本科线只在源一分一段里"路过"过（被解析丢弃，现已采）；接新省要确认该省源也带「控制线」列，否则该省
  本科线会空。源只有本科线、没有特控线——若将来要展示特控线，得另接数据源（与单一源原则权衡）。
