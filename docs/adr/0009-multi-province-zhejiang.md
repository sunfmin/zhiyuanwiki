# 多省份泛化与浙江接入（综合单科类 + 专业平行志愿）

ADR-0002 当初承诺"加省份只是加数据、不是改架构"，但首版把**单一省份**与**物理/历史双科类**写死在了数据布局、Go 管线、前端路由与组件里。接入浙江暴露出浙江高考模型与黑龙江**结构性不同**，因此本次做了一轮多省份泛化。

## 浙江与黑龙江的结构差异

- **单科类「综合」**：浙江不分物理/历史，全省一套排名。`schools.json` 的 `wuli/lishi`、`locator-wuli/lishi.json`、组件里的双科类假设都需泛化为"按科类名索引"。
- **专业平行志愿**：填报单位是**院校×专业**（每个专业独立投档、**无组内调剂**），不是黑龙江的院校专业组。校页 2026 视图与定位项据此分两种填报模型（`group` / `major`）。
- **分段制**：普通类一段/二段（命名逐年变：旧年份「平行录取一段/二段」）+ 提前批，而非"本科批"。本站收录 一段+二段+提前批 全量普通类。
- **选科 7选3**：选考科目为物理/化学/生物/政治/历史/地理/技术 选 3；选科要求格式也不同（"物化生(3科必选)"、"物/化/生(3选1)"、"思想政治必选"…），需要独立的选科匹配器。
- **数据更全更干净**：浙江源表内联了学校所在/性质/985/211/选科要求；「一表联动」表（66 列）直接给出门类、城市水平标签（=城市层级，无需静态映射）、院校标签（解析 985/211/双一流）、公私性质、类型、软科排名。院校代码为 4 位定宽稳定键（优于黑龙江的按校名挂接），但 2026 计划表把代码存成数字丢了前导 0，导入时统一补零对齐。

## 决定

1. **共享原语下沉到 `internal/core`**：专业名键、行表聚合、一分一段换算、等效位次、门类分类器、xlsx 助手、院校代码归一等省份无关逻辑集中一处；`internal/hlj`、`internal/zj` 各自只放省份专属解析。黑龙江原有测试为安全网，重构后产物逐文件 byte 对齐校验。
2. **数据按省分目录**：`src/data/<slug>/…`、`public/data/<slug>/…`（slug=hlj/zj）。`schools.json` 索引项的 `wuli/lishi` 泛化为 `ranges{科类名→区间}`；校详情用判别字段 `groups2026`（黑龙江）/ `plan2026`（浙江）。
3. **预处理工具加 `-prov`**：`go run ./cmd/zhiyuan-data {fenduan,yuanxiao,zhuanye,dingwei} -prov hlj|zj`，按省选解析器与输出目录。
4. **前端对称 `/[prov]/` 路由**：`/hlj/…`、`/zj/…`；根 `/` 跳默认省（黑龙江），旧单省 URL 由 `public/_redirects` 在 CF 边缘 301。`src/lib/provinces.ts` 是省份配置（科类、填报模型、选科模型、文件名片段）的前端真相源，与 `cmd/.../provinces.go` 镜像。
5. **组件按省配置泛化**：定位器按 `tracks` 决定是否显示科类切换、按 `subjectMode` 切换"首选+再选 / 7选3"、按 `fillModel` 决定是否显示组内调剂提示；选科匹配、localStorage、数据 fetch 路径、站内链接均按省命名空间/前缀。

## 代价与边界

- 黑龙江 `internal/hlj` 用类型别名再导出 core 符号以保持调用方与测试不变——多一层薄门面，但逻辑只此一份。
- 浙江等效位次目标年取 2025（最近有一分一段的年份；2026 高考未考、无一分一段），与黑龙江等效到 2026 口径不同，校页与定位文案据省显示。
- 本次**不含**浙江艺术/体育/专项计划，及其余 29 省（顶栏切换仍提示"暂未开放"）。
