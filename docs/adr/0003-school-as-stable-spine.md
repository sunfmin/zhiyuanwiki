# 数据模型主干：院校为稳定实体，院校×专业为叶子页，院校专业组只作单年视图

wiki 的常驻、可互链页面以**院校**（稳定校代码）为主干，以**院校×专业**为数据叶子页（承载历年专业录取分数线/位次/选科走势）。

**院校专业组不做永久实体**：它的组号**逐年变**（例：东北财经 2025 物理 008 组，2026 改为 009–018 组），是单年的报考产物。若以它为主干，跨年链接会断、URL 不稳定。因此院校专业组只作为"某一年的报考视图"呈现在院校页之下；把往年专业分数线挂到当年组上时，**按"院校+专业名"等效挂接，绝不按组号直接套**。

这是最容易被后人"好心改错"的地方——看起来院校专业组才是平行志愿的填报单位、"理应"做成主干——故在此显式记录：稳定实体是院校与专业，组只是视图。
