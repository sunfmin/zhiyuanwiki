# 院校专业组按 (院校,科类,组代码) 一等，往年位次按科类挂接

接入四川时发现 `core.BuildGroups2026` 把招生计划按 `(院校, 组代码)` 聚合成院校专业组，**键里漏了
科类**。多数省没暴露问题：它们同校的物理组与历史组用不相交的组代码。但四川/安徽等省**在两科类间
复用同一组代码**（同校物理「101」组与历史「101」组各自存在），于是两组被并成一个，组的科类取首行
（物理先到→整组标物理），历史专业被吞进物理组。后果：四川历史定位单位从应有的数千塌成 84 个。

这其实是**所有 group 省的潜在串档**：江苏 615、湖南 1630、四川 1926、安徽 1315 个 (校,组代码) 对
被两科类共用（只是江苏/湖南的碰撞多在老年份，而组视图只取最新计划年，故肉眼未察觉）。

第二个相关缺陷：组内专业挂接往年最低位次用的是 `leafLatest`（**全科类**最近点）。同名专业在物理/历史
都有线时，历史组的专业会错挂到物理位次——定位按位次分「冲稳保」，串了科类就分错档。

## 决定

1. `BuildGroups2026` 的聚合键改为 `(院校, 科类, 组代码)`；组内排序改为按 (科类, 组代码)，保证确定性
   （同号跨科类两组顺序稳定→ JSON 可复现）。
2. 新增 `leafLatestForTrack(leaf, track)`：优先取该**科类**下最近年份的往年线，该科类无线再回退
   `leafLatest`（3+1+2 下同名专业可能仅另一科类有往年线，回退好过完全无位次）。组内专业挂接改用它。

院校专业组本就是按科类报考的一等单位（「南大物理第05组」与「南大历史第05组」是两个组），此前的键
是 bug，不是简化。

## 影响

- 修复四川历史（84→3184 定位单位）、安徽（历史去污后 ~2411），并校正所有 group 省。
- **改动了已提交数据**：江苏（+6 组）、黑龙江（净行数不变、仅值/序变化）——均为更正而非回归；
  浙江（`major` 模型，不经此函数）字节不变，是回归护栏。
- `internal/core/groups_test.go` 固化跨科类拆分 + 按科类挂接位次两条不变量。

## 给未来的人

任何「按组聚合」的逻辑，键都要带科类。新省若复用跨科类组代码，本修复已兜住；若某省连组代码都没有
（重庆/贵州/辽宁，见 ADR-0014），那不是本函数能救的——它需要 `major` 模型而非 group。
