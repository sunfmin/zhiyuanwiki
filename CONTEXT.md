# 高考志愿数据 Wiki（黑龙江先行）

帮助高考考生用历年官方数据选择院校与专业的**公开 wiki**。本文件是术语表（统一用词），不放具体方案与实现细节——方案性的关键决定记在 `docs/adr/`。

## Language — 录取数据

**位次**：
考生在本类别（物理类/历史类）内的全省累计排名。报志愿的**核心参照**——比分数稳定，因为题目难易每年浮动，但"排第几名能上哪所学校"相对恒定。本站一切定位、筛选、走势均以位次为准。
_Avoid_：名次、排名、分数（分数是表象，位次才是依据）

**一分一段表**：
省招生考试院发布的"每个分数及其以上累计人数"对照表，用来把分数换算成位次。每个（省份×科类×年份）一张。

**等效位次**：
把往年某校某专业的录取位次，放到今年来看大约相当于多少分/什么名次，从而跨年份比较"今年够不够得着"。本站做跨年比较的唯一正确方法。
_Avoid_：等效分（口语可用，但站内统一以位次表达）

**投档线**：
某**院校专业组**当年的投档最低分及其对应位次（平行志愿按组投档）。来源为官方公开数据。

**专业录取分数线**：
某**院校×专业**当年的录取数据——最低分、最低位次、最高分、选科要求。是叶子页最丰富的数据。
_Avoid_：专业分数线（同义，站内统一用"专业录取分数线"）

**招生计划**：
某年某院校专业组下逐专业的计划招生人数、学制、学费、选科要求、专业全称/备注。是"今年能填什么"的真相来源。

## Language — 主体实体

**院校**：
一所学校，由稳定的校代码标识。**本站的主干稳定实体**——每所院校一个常驻页面，聚合其历年投档线/位次走势、招生计划与开设专业。
_Avoid_：大学、学校、高校（站内统一用"院校"）

**专业**：
一个学科方向（如"计算机科学与技术"）。概念跨年稳定，可跨校横向对比。

**院校×专业**：
某所院校开设的某个专业。**本站的数据叶子页**——承载该(院校,专业)的历年专业录取分数线与选科要求走势。

**院校专业组**：
平行志愿的实际填报单位：一所院校内若干专业打包成一组，共享同一套选科要求与"是否服从调剂"。**组号逐年变（是单年产物，不是跨年稳定实体）**，故本站只把它作为"某一年的报考视图"呈现在院校页之下，不做永久可互链的实体页。
_Avoid_：专业组、招生组（站内统一用"院校专业组"）

## Language — 填报与定位

**平行志愿**：
同一批次可填多个志愿，按"位次优先、遵循志愿顺序"投档，每人只投一次。意味着志愿表的**顺序**很关键。

**冲 / 稳 / 保**：
按录取把握把候选分三档——**冲**（录取位次略好于本人、搏一把）、**稳**（与本人相当）、**保**（明显差于本人、确保有学上）。本站的客户端定位功能据此对访客分档。

**服从专业调剂 / 组内调剂**：
所报专业满额时，是否同意被调到**同一院校专业组内**的其他专业（只在组内，不跨组）。勾"服从"大幅降低退档风险，代价是可能被调到不喜欢的专业——故选组时要看组内有没有绝对不想读的专业。

**选科 / 物理类 / 历史类**：
新高考 3+1+2 下首选物理或历史的考生类别。两类**分开排名、分开划线、分开录取**，同一分数含金量不同；位次、一分一段、投档线均按类别分别计算。
_Avoid_：理科/文科（旧高考叫法，已不准确）

**选科要求**：
某院校专业组/专业对再选科目的限定（如"不限""化学""化学和生物"）。决定一个考生的选科组合能不能填某个组。

**访客位次**：
访客在浏览器本地输入的自己的位次（及科类/选科），仅在客户端用于筛选与冲稳保分档，**不上传、不存储**。
